# Data Dictionary

Schema of the synthetic Northwood Hospitality Group dataset that powers the
live demo. All data is generated by `server/scripts/generate-data.js` with a
deterministic seed.

## customers.json (guests)

| Field                | Type      | Notes |
|----------------------|-----------|-------|
| `customer_id`        | string    | `GST-#####` |
| `first_name`         | string    | Generic neutral list |
| `last_name`          | string    | Generic mixed list |
| `email`              | string    | Synthetic, never a real address |
| `phone`              | string    | Format `(231) 555-1212` |
| `age`                | int       | Tavern skews younger, resorts skew older |
| `home_store_id`      | string    | `STR-XXX` |
| `home_store`         | string    | Display name (Wrightsville Beach Resort, Bald Head Island Club, ...) |
| `acquired_at`        | date      | YYYY-MM-DD |
| `loyalty_enrolled`   | bool      | Per-property enrollment rate |
| `opt_in_email`       | bool      | ~68% chain-wide |
| `opt_in_sms`         | bool      | ~51% chain-wide |
| `opt_in_push`        | bool      | ~38% chain-wide |
| `annual_spend`       | float     | Trailing 12 months |
| `lifetime_spend`     | float     | Sum of all folios |
| `visits_90d`         | int       | Trailing 90-day visit count |
| `last_visit`         | date      | YYYY-MM-DD |
| `days_since_last`    | int       | Days since last folio |
| `avg_basket`         | float     | Lifetime folio average |
| `tier`               | enum      | `Silver` / `Gold` / `Platinum` |
| `churn_risk`         | float     | 0.0 to 0.99 |
| `rising_star`        | bool      | 118 guests true |
| `decliner`           | bool      | 203 guests true |
| `near_platinum`      | bool      | 89 guests true |
| `vip_likely`         | bool      | Bald Head Island Club and Southport Waterfront Lodge heavy |
| `favorite_category`  | string    | Most-purchased category |
| `category_diversity` | int       | Distinct categories purchased |

## transactions.json (folios)

| Field            | Type   | Notes |
|------------------|--------|-------|
| `transaction_id` | string | `FOL-######` |
| `customer_id`    | string | FK to customers |
| `store_id`       | string | FK to stores |
| `store`          | string | Display name |
| `date`           | ISO    | UTC datetime |
| `items[]`        | array  | `{ category, qty, unit_price }` |
| `item_count`     | int    | Length of items |
| `subtotal`       | float  | Pre-discount |
| `discount_pct`   | float  | 0 to 0.25 |
| `total`          | float  | After discount |
| `channel`        | string | `email` / `sms` / `push` / `organic` |

## campaigns.json

| Field         | Type   | Notes |
|---------------|--------|-------|
| `campaign_id` | string | `CMP-#####` |
| `customer_id` | string | FK |
| `program`     | string | One of 10 program names |
| `channel`     | string | `email` / `sms` / `push` |
| `sent_at`     | ISO    | Send timestamp |
| `opened`      | bool   | Open event |
| `clicked`     | bool   | Click event |
| `redeemed`    | bool   | Redemption event |
| `revenue`     | float  | Attributed revenue (0 if not redeemed) |

## experiments.json

Pre-loaded list of two experiments, one completed (Platinum Double Points
Weekend, +19% lift, p=0.014) and one running (Resort Property Personalization
push test). See `generate-data.js` for the exact structure.

## holdouts.json

Pre-loaded permanent 5% marketing holdout used by the Holdout Manager page.

## stores.json (properties)

The 9 property profiles, their avg folio value, churn rate, share of guest
base, loyalty enrollment rate, and property-type tag (`urban_hotel`,
`luxury_resort`, `boutique_hotel`, `casual_dining`, `fine_dining`, `spa`,
`event_venue`, `boutique_inn`).

## PMS, POS, and loyalty field mapping

The Data Manager page in the demo shows the canonical mapping between our
field names and common hospitality-system field equivalents (PMS, POS, and
loyalty platform). That mapping is hard-coded in
`client/src/demo/DataManager.jsx` and is illustrative, every stack has its
own quirks.
